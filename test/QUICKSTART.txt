Getting Started with TextAdventurer Test Environment
======================================================

You now have a complete **standalone test environment** for TextAdventurer! 

This lets you:
✅ Test addon commands without restarting WoW
✅ Iterate in milliseconds instead of minutes
✅ Run command routing, calculations, and logic validation
✅ Use interactive REPL or automated batch testing
✅ Extend with custom test data and cases

═════════════════════════════════════════════════════════════════════

QUICK START (5 minutes)
═════════════════════════════════════════════════════════════════════

1. Install Lua 5.1:

   OPTION A: Automated Setup (Recommended)
   ───────────────────────────────────────
   Open VS Code command palette (Ctrl+Shift+P) and run:
   > Setup Test Environment
   
   This will check for Lua and offer to install it.

   OPTION B: Manual Install
   ─────────────────────────
   Windows: choco install lua
   macOS:   brew install lua
   Linux:   apt-get install lua5.1

2. Verify Installation:
   
   Windows (in addon folder):
   test\setup.bat
   
   macOS/Linux:
   bash test/setup.sh

3. Run Tests:

   Interactive Mode (Type commands in a REPL):
   lua test/test-runner.lua . interactive
   
   Batch Mode (Run predefined tests):
   lua test/test-runner.lua . batch

═════════════════════════════════════════════════════════════════════

VS CODE INTEGRATION (Easiest)
═════════════════════════════════════════════════════════════════════

If using VS Code, you can run tests with one command:

1. Press Ctrl+Shift+P (Command Palette)

2. Choose one:
   → "Setup Test Environment"       (Install Lua + verify)
   → "Run Interactive Test Console" (REPL mode)
   → "Run Batch Tests"             (Automated testing)

WHAT YOU CAN TEST
═════════════════════════════════════════════════════════════════════

✅ Command Routing
   - Test exact handlers (health, stats, gear, etc.)
   - Test pattern handlers (help topics, action bar ranges, etc.)
   - Verify error messages for unknown commands

✅ Data Lookups
   - GetItemInfo() - item properties, rarity, sell value
   - UnitHealth() - unit stats and vitals
   - GetSpellInfo() - spell names and icons
   - C_Container APIs - inventory management

✅ Calculations
   - Warrior Prompt formulas
   - Paladin Seal DPS calculations
   - Warlock DPS calculations
   - Stat aggregation and formatting

✅ UI Logic
   - Command output formatting
   - Multiline input with comments
   - Block replay (runlast/rerun)
   - Tab completion cycling
   - Help system and topic filtering

✅ Edge Cases
   - Invalid inputs
   - Range validation
   - Empty results
   - Boundary conditions

═════════════════════════════════════════════════════════════════════

LIMITATIONS
═════════════════════════════════════════════════════════════════════

The mock environment is FUNCTIONAL but NOT REALISTIC:

Can Test:
✅ Command routing and handlers
✅ Data lookups and calculations
✅ String formatting and output
✅ Logic branches and state
✅ Error messages

Cannot Test:
❌ Real UI rendering
❌ Protected WoW actions (CastSpellByName)
❌ Real event firing
❌ Texture/font rendering
❌ Network operations

For UI/rendering validation, still need in-game testing.
